### Event Tracking (Webhooks)
Resend sends webhooks for:
- âœ… `email.sent`
- âœ… `email.delivered`
- âœ… `email.opened`
- âœ… `email.clicked`
- âœ… `email.bounced`
- âœ… `email.complained` (spam report)

Our webhook handler inserts to `outreach_events`:

```javascript
// Webhook handler (frontend/api/webhooks/resend.ts)
export async function POST(request: Request) {
  const payload = await request.json();
  
  // Verify webhook signature
  const signature = request.headers.get('resend-signature');
  if (!verifySignature(payload, signature)) {
    return new Response('Invalid signature', { status: 401 });
  }
  
  // Insert event
  await supabase.from('outreach_events').insert({
    contact_id: payload.data.tags.find(t => t.name === 'entity_id').value,
    campaign_id: payload.data.tags.find(t => t.name === 'campaign_id').value,
    event_type: payload.type.replace('email.', ''),
    event_data: payload.data,
    resend_email_id: payload.data.email_id,
    occurred_at: payload.created_at,
  });
  
  return new Response('OK');
}
```

This automatically updates:
- `contacts.email_opens`, `contacts.last_opened_at`
- `contacts.email_clicks`, `contacts.last_clicked_at`
- `contacts.outreach_status` â†’ 'engaged' on click
- `outreach_campaigns.emails_opened`, etc.

## ðŸš¨ Risk Mitigation

### Risk 1: Poor Deliverability

- Slow domain warm-up

**Mitigation:**
- A/B test aggressively (10+ subject line variations)
- Segment hygiene (remove unengaged after 90 days)
- Re-engagement campaigns before suppression
- Offer genuine value (free audit is not bait)

### Risk 4: Resend Account Suspension
**Mitigation:**
- Stay under sending limits
- Monitor bounce/spam rates obsessively
- Have backup provider ready (SendGrid, Postmark)
- Keep suppression list synced across providers
